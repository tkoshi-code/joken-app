# さいこー雀 — 開発エージェント向けガイド

麻雀条件戦の条件計算・成績記録PWA。オーナーは最高位戦の現役プロ(D3リーグ所属)。
仕様の詳細は SPEC.md、機能一覧は README.md を参照。

## 構成

- **index.html 単一ファイルアプリ**(バニラJS、フレームワーク・ビルドなし) + sw.js + manifest.json
- データはすべて端末内: localStorage(キー `jokenapp_v1`、状態オブジェクト `S`)+ IndexedDB(局面メモの写真)
- `data/shared.json` = 全ユーザー共通の配信データ(公式日程)。読み取り専用、更新はこのファイルを直接編集してpush

## 変更時の必須ルール

1. **機能追加・修正のたびに必ず3点セットで更新**:
   - `index.html` の `APP_VERSION`(例 '1.28' → '1.29')
   - `CHANGELOG` 配列の先頭に `['バージョン','YYYY-MM-DD','変更内容']` を追加
   - `sw.js` の `CACHE`(例 'joken-v32' → 'joken-v33')
2. ユーザー向けの機能を変えたら **アプリ内の使い方ガイド(vHelp)** と README.md も更新する
3. コミットメッセージは日本語、末尾にバージョン `(v1.XX)` を付ける

## デプロイ

- `main` に push すると GitHub Pages(https://tkoshi-code.github.io/joken-app/)に1〜2分で自動反映
- Service Worker は HTML と shared.json をネットワーク優先(3秒でキャッシュfallback)、他はキャッシュ優先。
  ユーザーはオンラインで開き直すだけで最新版になる

## 動作確認

- ローカル: `npx -y http-server . -p 8321 -c-1` → http://localhost:8321
- 計算ロジックを触ったら必ず手計算と突き合わせる(ポイントは内部で0.1P単位の整数=tenths、`fmtP`/`fmtPt`で表示)

## ドメイン知識(壊してはいけない仕様)

- ルールプリセット: 最高位戦(30000持ち返し・ウマ+30/+10/-10/-30・オカなし)、Classic(同・±12/±4)、
  Mリーグ(25000持ち30000返し・オカ+20・切り上げ満貫・同点は起家順)
- 半荘内同点: 最高位戦/Classicは順位点折半(着順2.5等の0.5刻み)。供託は最高位戦では場から除外(トップ取りしない)
- 素点入力の検算: 4人合計 = 配給原点×4 − 供託。ポイント入力の検算: 4人合計 = 0
- 条件表・オーラス判定は実在する点数(点数表)のみで逆算する(存在しない点数を出さない)
- 順位表のパーサーは3形式対応: HP形式(Total列で検算)/ X画像形式(ポイント=開始時点+節TOTAL)/ 手入力「名前 12.3 ▲5.0」

## データ構造(localStorage `S`)

- `players` プレイヤー帳 / `events` 大会(mode: tournament|league|table|border) / `myGames` 成績タブの個人記録
- `videos` 動画ストック / `situations` 局面メモ / `schedule` 手動予定 / `myMark` 公式予定の参加キー
