# 条件戦サポートアプリ 仕様書 (v0.1 ドラフト)

最高位戦の条件戦（トーナメント・ワンデー・リーグ戦）で、半荘ごとのスコアを正確に記録し、
最終半荘前に「通過条件」を計算ミスなく算出するスマホアプリ。

- 形態: **PWA**（ブラウザで開いてホーム画面に追加、完全オフライン動作）
- データ: 端末内保存（localStorage/IndexedDB）+ JSONエクスポート/インポート
- 運用前提: 対局卓にスマホは持ち込めないため、**最終半荘前に条件表を紙に転記する**。
  転記しやすさを最優先した「転記用ビュー」を用意する。

---

## 1. ルールプリセット

ポイント計算に効くパラメータのみ保持する（一発・裏ドラ等の打牌ルールは対象外）。

| パラメータ | 最高位戦 | Classic | Mリーグ |
|---|---|---|---|
| 配給原点（持ち点） | 30,000 | 30,000 | 25,000 |
| 返し点 | 30,000 | 30,000 | 30,000 |
| オカ | なし | なし | あり（トップ+20P） |
| 順位点 | +30/+10/−10/−30 | +12/+4/−4/−12 | +30/+10/−10/−30 |
| ポイント単位 | 0.1P（100点単位） | 同左 | 同左 |

- 計算式: `ポイント = (素点 − 返し点) ÷ 1000 + 順位点 (+ オカ)`
- **カスタムプリセット**: 返し点・順位点・オカを自由設定して保存可能（他団体戦への拡張用）
- 半荘内同点時の順位点: **最高位戦・Classicは折半（確定）**。2位と3位が同点なら (+10−10)÷2=0ずつ、着順は2.5として記録。
  Mリーグプリセットは「起家に近い方が上位」を既定とする（プリセット項目として切替可）
- 供託残りの扱いもプリセット項目: **最高位戦系は供託は別（トップ取りしない）**、他団体ルールではトップ取りがあり得るため切替可

## 2. モード

### 2.1 トーナメントモード（同卓内で完結）

- 大会設定: ルールプリセット / 回戦数（2〜5） / 通過人数（1位抜け・2位抜け・3位抜け）/
  同点時の扱い（先着有利 / 追いつき有利）
  - 先着有利 = トータル同点なら最終半荘開始時点で上位だった者が上位（確定）
- 同卓4名を選択（プレイヤー帳から。新規名はその場で登録）
- 半荘ごとにスコア入力 → 累計・現順位・通過圏との差を常時表示
- 最終半荘前に**条件表**を生成（§4）

### 2.2 ボーダー比較モード（ワンデー・リーグ戦、卓またぎあり）

- 全体の通過ライン（ボーダー推定値）を手入力
- 自分の累計との差分、最終戦で必要なポイント（必要着順+素点の目安）を表示
- ボーダーは対局中に動くため、複数のボーダー想定値（楽観/悲観など）を並べて条件を出せる

## 3. スコア入力

**素点入力とポイント入力を対等にサポート**（大会ごと・半荘ごとに切り替え可）。

- 素点入力: 終局時の持ち点を4人分入力（**入力順は起家からの席順**とする）
  - 検算: 4人合計 = 配給原点×4 − 供託残（供託残は別欄で入力。トップ取りルールの場合はトップの素点に自動加算）
  - ウマ・オカ・順位点・同点折半はアプリが自動計算
- ポイント入力: 計算済み±ポイントを4人分入力
  - 検算: 4人合計 = 0（オカ・供託分のズレは警告表示）
- 入力ミス時の修正: 過去の半荘をいつでも編集可、累計は自動再計算

## 4. 条件表（最終半荘前）

対象者（既定は自分）について:

1. **現況**: 各者の累計ポイント・順位・通過圏との差（順位順）
2. **着順マトリクス**: 自分の着順 × 競争相手の着順 の全組み合わせについて、
   順位点の跳ね返りを織り込んだ「通過に必要な素点差」を100点単位で表示
   - 例:「自分2着・A氏3着 → 素点差12,300点以内なら通過」
3. **文章サマリー**: 「2着以上で無条件通過」「3着なら A氏が4着かつ素点差○○点以内」等
4. **転記用ビュー**: 上記を1画面に圧縮した手書き転記向けレイアウト
5. 同点時設定（先着/追いつき）を境界値の「以上/超」に正確に反映する

※ 打点条件への換算は §4.5 オーラス判定として実装済み（2026-07-07）。

## 4.5 オーラス判定（最終局の打点条件）

最終局開始時点の状況（4人の現在素点・供託・本場・親）を入力すると、
実在する点数（3,900/7,700/満貫…切り上げ満貫ルールにも対応）を全て当てはめて
半荘精算→累計→着順（トータル同点は先着/追いつき設定）までシミュレーションし、以下を表示:

- **アガリ条件**: ツモの最低打点、ロンは放銃者別の最低打点（直撃効果を正確に反映）
- **通過順位切替**: 1位通過〜通過人数まで切り替えて表示
- **相手のアガリ許容**: 相手ごとに ツモられ/自分の放銃/横移動 の「ここまでなら通過維持」上限
- **流局判定**: テンパイ料（場3,000）の全組み合わせ（8通り×自分のテンパイ有無）で◯×表
  - ノーテン罰符なしのルール（Classic）は罰符移動なしで単一判定
- この局で終局する前提（連荘・途中流局は対象外）。リーチ棒は素点から引いて供託に加算して入力

### 2.3 リーグ戦(記録)モード（2026-07-07 追加）

- リーグ戦・研究会などの成績記録用。半荘を追加していく形式（回数無制限）
- 同卓メンバーは半荘ごとに選択（その場で新規プレイヤー登録も可）。日付（=節）を各半荘に記録
- 大会画面: 自分の累計・平均着順・着順分布（同点折半の2.5着は0.5回ずつ集計）、
  **ポイント推移グラフ**（SVG折れ線、自分/全員/任意の相手を切替）、節ごとの対局記録リスト
- 半荘の削除・修正可（削除時は半荘番号を自動で振り直し）
- 記録はプレイヤー帳の対戦履歴・通算成績にも反映

### 2.4 リーグ順位表モード（2026-07-07 追加）

- 公式HPのリーグ成績表（順位/選手名/Total/地域/対局数/節ポイント、▲=マイナス）を
  **コピー&ペーストで取り込み**。Total列は節合計との検算に利用し、列構成を自動判別
- 手入力は「名前 12.3 ▲5.0」形式の行でも取り込み可（Xの結果からの入力用）
- 取り込みモード: 全節上書き（HP貼り付け用）/ 節を追加（節単位の手入力用）
- 全体順位表（自分をハイライト・昇級ライン差または自分との差）
- ライン設定は3種: **2段階昇級（上位N位）/ 昇級（上位N位）/ 降級（下位N名）**
  - 降級ラインは「残留できる最下位のTotal」を基準に差を表示
  - 順位表は圏内を色分け（2段階昇級=紫 / 昇級=緑 / 降級=赤）、各ラインとの差を自分の行に表示
  - 推移グラフに3ラインとも破線で描画（紫/赤/グレー）
- 節ごとの累計推移グラフ（自分+昇級ライン+比較相手を選択可）
- 選手名は文字列として保持（プレイヤー帳とは独立、数百名規模に対応）

## 5. プレイヤー帳（対局者の蓄積）

- 項目: 名前・読み仮名・所属・メモ
- 所属は固定マスタで管理: **最高位戦 / 連盟 / 協会 / RMU / 連合 / 101 / その他 / 一般**
- スコア入力時にオートコンプリート
- 対局結果は自動でプレイヤーに紐付け、対戦履歴（対局数・平均着順・直近対局）を参照可能

## 6. データモデル（概要)

- `players`: id, 名前, 読み, 所属, メモ
- `rulePresets`: 名称, 返し点, 順位点[4], オカ, 同点時処理
- `events`: 名称, 日付, モード, プリセット, 回戦数, 通過人数, 同点設定(先着/追いつき)
- `games`: eventId, 回戦, 参加4名, 素点[4] or ポイント[4], 供託残
- `borders`: eventId, ボーダー想定値（複数可）

## 7. フェーズ分け

- **フェーズ1**: プリセット3種+カスタム / トーナメントモード / 素点・ポイント入力 /
  条件表（着順×素点差） / 転記用ビュー / プレイヤー帳 / エクスポート
- **フェーズ2**: ボーダー比較モードの強化（複数想定値）/ 打点条件換算 / 対戦成績の統計表示

## 8. 確定事項（2026-07-07 ユーザー確認済み）

1. 半荘内同点時の順位点は**折半**（最高位戦・Classic。2位3位同点なら0ずつ、着順2.5）
2. 先着有利/追いつき有利 = 最終半荘開始時点の上位者優先/その逆（確定）
3. 供託は最高位戦では**別扱い（トップ取りしない）**。他団体ルール用にトップ取りも選択可
4. トビペナルティなし（箱下続行）
5. 所属マスタ: 最高位戦/連盟/協会/RMU/連合/101/その他/一般

### 残る軽微な確認事項

- Mリーグプリセットの既定値（同点=起家順、供託=トップ取り）が正しいかは使用時に要確認
- PWAの配置先（GitHub Pages 等の HTTPS ホスティング推奨。オフライン化に必要）
