# さいこー雀

麻雀条件戦の条件計算・成績記録を行うスマホ向けPWA(旧称: 条件戦サポート)。
詳細仕様は [SPEC.md](SPEC.md) を参照。バージョン履歴はアプリ内(設定 → バージョン履歴)。

**公開URL: https://tkoshi-code.github.io/joken-app/**

## 画面構成

- **ホーム = トップメニュー**: 最近の大会 + 各機能への入口(それぞれ1行説明付き)。使い方ガイドへもここから
- 下部ナビ: ホーム / 成績 / プレイヤー / 設定

## 主な機能

### ルールプリセット
- 最高位戦 / 最高位戦Classic / Mリーグ + カスタムルール(返し点・順位点・オカ・同点処理・供託・ノーテン罰符・切り上げ満貫を自由設定)
- ポイント計算はアプリが自動で行い、素点入力時は合計点の検算で入力ミスを検出

### トーナメント(2〜5回戦・1〜3位抜け)
- 半荘ごとのスコア入力(素点/ポイント両対応)、累計と現順位を常時表示
- **条件表**: 最終戦前に「自分×相手の着順ごとの必要素点差」マトリクスを生成。
  同点は先着有利/追いつき有利の設定を反映。紙への転記用の白黒表示(転記モード)付き
- **オーラス判定**: 最終局前の持ち点・供託・本場・親を入力すると、実在する点数だけを使って
  「ツモ○○以上」「誰々から直撃○○以上」「放銃は○○まで」「流局時のテンパイ組合せ判定」を算出。
  1位通過〜N位通過の条件を切り替え表示

### 対局記録(リーグ戦・研究会など)
- 半荘を追加していく記録モード。同卓メンバーは半荘ごとに選択、日付(節)ごとにグループ表示
- 累計・平均着順・着順分布(同点折半は0.5回集計)と、半荘ごとの累計推移グラフ(自分/全員切替)

### 成績タブ(個人の通算記録帳)
- 大会と独立して自分の対局を蓄積。**タグ(リーグ・カテゴリ)ごとのカード表示**で、タップすると成績・推移グラフ・履歴
- **半荘ごとにルールを選択可**、フリーメモ欄付き。「4人分記録」と「自分のみ記録(着順+ポイントだけ)」を選択可
- 「まとめて取り込み」: 「日付 着順 ポイント」形式の行を貼り付けて一括登録(Xの結果画像からの転記用)

### 順位表(リーグ全体の動向)
- **公式HPの成績表をコピー&ペーストで取り込み**(▲マイナス・全角数字対応、Total列で検算)。
  「名前 12.3 ▲5.0」形式の手入力行もOK。全節上書き/節を追加の2モード
- **当日のX結果画像**: スマホのテキスト認識(iPhoneライブテキスト/Googleレンズ)でコピーした行を自動判別して
  節TOTALのみ抽出。または「節を手入力」(名前一覧に数字を打つだけ)
- 2段階昇級(上位N位)/ 昇級(上位N位)/ 残留(上位N位)の3ライン設定 → 各ラインとの差表示・圏内色分け
- 節ごとの累計推移グラフ: 自分+3ライン(破線)+比較相手 最大4人(色分け)。比較相手はプレイヤー帳へワンタップ登録可

### ボーダー比較(ワンデーなど)
- 自分のトータルとボーダー想定値(複数登録可)から、着順ごとの必要素点を表示

### スケジュール
- 予定の追加(「日付 [時刻] タイトル [@場所]」形式の一括追加も可)。大会の日程(リーグの複数開催日含む)は自動表示
- **公式日程(東京)を共有データとして配信**(`data/shared.json`、「公式」バッジ・リストは直近60日分)
- **リスト/月カレンダー切替**。カレンダーは色分け(緑=自分/青=大会/灰=公式)、日付タップで詳細・その日への予定追加
- フィルター(すべて/自分の予定/大会/公式+キーワード検索)
- 公式予定に**「☆参加」マーク** → 自分の予定として扱われ、期間制限に関係なく常に表示

### 動画ストック
- 見た動画をURL+タイトル・チャンネル(URLから自動取得)+タグ+学びメモで蓄積

### 局面メモ
- 牌パレットまたは記法(234m567p88s東東、赤5=0m)で牌姿を入力、SVG牌画プレビュー
- 写真添付(端末内IndexedDBに保存)、状況・論点メモ
- **1枚のPNG画像に合成して共有**(牌画+写真+質問、LINE/X等の共有シートへ)

### プレイヤー帳
- 名前・所属(最高位戦/連盟/協会/RMU/連合/101/その他/一般)・メモを登録
- 対局記録から通算成績(対局数・平均着順・通算P)と対戦履歴を自動集計

## スマホへの導入

1. https://tkoshi-code.github.io/joken-app/ をスマホのブラウザで開く
2. **ホーム画面に追加**
   - iPhone: Safariで開き、共有ボタン →「ホーム画面に追加」
   - Android: Chromeで開き、メニュー →「ホーム画面に追加」
3. 一度開けば以後は**オフラインでも動作**します(会場に電波がなくてもOK)

アプリの更新は自動です(オンラインで開けば常に最新版。オフライン時はキャッシュで起動)。

## データについて

- データは端末のブラウザ内(localStorage)にのみ保存され、サーバーには一切送信されません
- **設定 → バックアップ(JSON書き出し)** で定期的にバックアップしてください(節の終わりごと推奨)
- 機種変更時は書き出したJSONを新端末で「復元」してください

## 開発メモ

- 構成: `index.html`(単一ファイルアプリ) + `sw.js`(Service Worker) + `manifest.json`
- ローカル確認: `npx -y http-server joken-app -p 8321 -c-1` → http://localhost:8321
- デプロイ: `main` ブランチに push すると GitHub Pages に自動反映(1〜2分)
- Service Worker はHTMLのみネットワーク優先(3秒でキャッシュにフォールバック)、
  静的ファイルはキャッシュ優先。更新時は `sw.js` の `CACHE` バージョンを上げること
